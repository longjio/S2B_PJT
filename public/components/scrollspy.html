First heading

This is some placeholder content for the scrollspy page. Note that as you scroll down the page, the appropriate navigation link is highlighted. It's repeated throughout the component example. We keep adding some more example copy here to emphasize the scrolling and highlighting.

Bootstrap의 Scrollspy는 스크롤 위치를 기반으로 네비게이션 링크를 자동으로 업데이트하는 강력한 기능입니다. 사용자가 페이지를 아래로 스크롤하면 현재 뷰포트에 있는 섹션에 해당하는 네비게이션 링크가 자동으로 활성화됩니다.

이 기능은 긴 문서나 단일 페이지 애플리케이션에서 특히 유용합니다. 사용자는 현재 어느 섹션을 보고 있는지 쉽게 파악할 수 있으며, 네비게이션을 클릭하여 원하는 섹션으로 빠르게 이동할 수 있습니다.

Second heading

두 번째 섹션입니다. Scrollspy는 data 속성을 사용하여 쉽게 설정할 수 있습니다. data-bs-spy="scroll" 속성을 스크롤 가능한 요소에 추가하고, data-bs-target 속성으로 네비게이션 요소를 지정하면 됩니다.

추가적으로 data-bs-root-margin 속성을 사용하여 활성화 시점을 조절할 수 있습니다. 이는 CSS의 root margin과 유사하게 동작하며, 요소가 뷰포트에 들어오는 시점을 정의합니다.

Scrollspy는 네비게이션 바, 사이드바, 목차 등 다양한 UI 패턴에 적용할 수 있습니다. Bootstrap의 nav, list-group, 또는 간단한 링크 목록과 함께 사용할 수 있습니다.

이 컴포넌트는 접근성을 고려하여 설계되었으며, 키보드 네비게이션과 스크린 리더를 지원합니다.

Third heading

세 번째 섹션입니다. Scrollspy를 사용할 때는 각 섹션에 고유한 ID를 부여해야 합니다. 네비게이션 링크의 href 속성은 이 ID와 일치해야 하며, 이를 통해 Bootstrap이 자동으로 스크롤 위치를 추적하고 링크를 업데이트합니다.

스크롤 컨테이너는 반드시 overflow-y: auto 또는 overflow-y: scroll 속성을 가져야 하며, 명시적인 높이가 설정되어 있어야 합니다. 또한 tabindex="0" 속성을 추가하여 키보드로 접근 가능하도록 만드는 것이 좋습니다.

Scrollspy는 중첩된 네비게이션도 지원합니다. 부모 항목과 자식 항목이 있는 계층적 네비게이션 구조에서도 올바르게 작동합니다.

JavaScript API를 통해 프로그래밍 방식으로 Scrollspy를 제어할 수도 있습니다. new bootstrap.ScrollSpy() 생성자를 사용하여 인스턴스를 만들고, refresh() 메서드로 업데이트할 수 있습니다.

이 예제에서 네비게이션 링크를 클릭하면 해당 섹션으로 부드럽게 스크롤되며, 전체 페이지가 아닌 컨테이너 내부만 스크롤됩니다.

<nav id="navbar-example" class="navbar navbar-expand-lg navbar-light bg-light">
  <div class="container-fluid">
    <a class="navbar-brand" href="#">Navbar</a>
    <nav class="nav nav-pills">
      <a class="nav-link" href="#scrollspy-1">First</a>
      <a class="nav-link" href="#scrollspy-2">Second</a>
      <a class="nav-link" href="#scrollspy-3">Third</a>
    </nav>
  </div>
</nav>
<div data-bs-spy="scroll" data-bs-target="#navbar-example" data-bs-root-margin="0px 0px -40%" style="height: 350px; overflow-y: auto;" tabindex="0">
  <h4 id="scrollspy-1">First heading</h4>
  <p>...</p>
  <h4 id="scrollspy-2">Second heading</h4>
  <p>...</p>
</div>

Item 1

중첩된 네비게이션을 사용한 Scrollspy 예제입니다. 이 구조는 문서의 계층적 구조를 반영하며, 부모 항목과 자식 항목 간의 관계를 명확하게 표현합니다.

메인 섹션 아래에 여러 하위 섹션이 있는 경우, 이러한 중첩 구조를 사용하면 사용자가 문서의 구조를 한눈에 파악할 수 있습니다.

Item 1-1

첫 번째 항목의 첫 번째 하위 섹션입니다. Scrollspy는 현재 스크롤 위치에 따라 부모 항목과 자식 항목을 모두 올바르게 하이라이트합니다.

중첩된 네비게이션에서는 Bootstrap의 nav-pills 클래스와 flex-column을 사용하여 세로 레이아웃을 구현합니다. ms-3 클래스로 하위 항목에 들여쓰기를 적용할 수 있습니다.

Item 1-2

첫 번째 항목의 두 번째 하위 섹션입니다. 각 하위 섹션은 독립적으로 추적되며, 사용자가 해당 섹션을 볼 때 자동으로 활성화됩니다.

이러한 구조는 API 문서, 사용자 가이드, 기술 문서 등에서 특히 유용합니다.

Item 2

두 번째 메인 항목입니다. 이 항목에는 하위 섹션이 없으므로 단독으로 표시됩니다.

Scrollspy는 복잡한 계층 구조를 처리할 수 있으며, 여러 레벨의 중첩을 지원합니다. 3단계, 4단계 이상의 중첩도 가능합니다.

네비게이션 구조를 설계할 때는 사용자 경험을 고려하여 너무 깊은 중첩은 피하는 것이 좋습니다.

Item 3

세 번째 메인 항목입니다. 이 항목도 하위 섹션을 포함하고 있습니다.

중첩된 Scrollspy를 구현할 때는 각 레벨의 ID 명명 규칙을 일관되게 유지하는 것이 중요합니다.

Item 3-1

세 번째 항목의 첫 번째 하위 섹션입니다. Bootstrap의 Scrollspy는 자동으로 부모-자식 관계를 인식하고 적절하게 활성화 상태를 관리합니다.

네비게이션 링크를 클릭하면 해당 섹션으로 부드럽게 스크롤되며, 전체 페이지가 아닌 스크롤 컨테이너 내부만 이동합니다.

Item 3-2

세 번째 항목의 두 번째 하위 섹션입니다. 이것으로 중첩된 네비게이션 예제가 완료됩니다.

실제 프로젝트에서는 이 패턴을 확장하여 더 많은 섹션과 하위 섹션을 추가할 수 있습니다. Scrollspy는 동적으로 추가되는 콘텐츠도 지원하며, refresh() 메서드로 업데이트할 수 있습니다.

List Group Scrollspy

Item 1 Item 2 Item 3 Item 4

Item 1

List Group을 사용한 Scrollspy 예제입니다. List Group은 Bootstrap의 강력한 컴포넌트로, 링크나 버튼의 목록을 깔끔하게 표시할 수 있습니다.

Scrollspy와 함께 사용하면 사이드바 네비게이션이나 목차를 만들기에 완벽합니다. list-group-item-action 클래스를 사용하면 호버 효과와 클릭 가능한 상태를 제공합니다.

현재 활성화된 항목은 자동으로 하이라이트되며, Bootstrap의 기본 스타일이 적용됩니다.

Item 2

두 번째 항목입니다. List Group Scrollspy는 특히 사이드바가 있는 문서 레이아웃에 적합합니다.

각 항목은 고유한 ID를 가지며, 네비게이션 링크의 href 속성이 이 ID를 참조합니다. Bootstrap의 Scrollspy JavaScript가 스크롤 위치를 모니터링하고 적절한 항목을 활성화합니다.

List Group은 다양한 스타일 옵션을 제공합니다. 기본, flush, horizontal 레이아웃 등을 선택할 수 있으며, contextual 클래스로 색상을 변경할 수도 있습니다.

이 컴포넌트는 반응형으로 설계되어 모바일 기기에서도 잘 작동합니다.

Item 3

세 번째 항목입니다. Scrollspy는 페이지가 로드될 때 자동으로 초기화되며, 추가 JavaScript 코드 없이 data 속성만으로 작동합니다.

동적으로 콘텐츠가 추가되거나 제거되는 경우, JavaScript API를 사용하여 Scrollspy를 새로고침할 수 있습니다. 이는 싱글 페이지 애플리케이션에서 특히 유용합니다.

List Group의 각 항목은 배지, 아이콘, 또는 추가 텍스트로 커스터마이즈할 수 있습니다. 이를 통해 더 풍부한 네비게이션 경험을 제공할 수 있습니다.

접근성 측면에서 List Group은 시맨틱 HTML을 사용하며, 스크린 리더와 키보드 네비게이션을 지원합니다.

Item 4

네 번째 항목입니다. 이것으로 List Group Scrollspy 예제가 완료됩니다.

실제 프로젝트에서는 이 패턴을 확장하여 더 많은 항목을 추가할 수 있습니다. 긴 문서의 경우 수십 개의 섹션도 문제없이 처리할 수 있습니다.

Scrollspy의 성능은 매우 우수하며, 복잡한 문서 구조에서도 부드럽게 작동합니다. Intersection Observer API를 기반으로 구현되어 있어 효율적입니다.

네비게이션 링크를 클릭하면 해당 섹션으로 스크롤되며, 스크롤 애니메이션은 smooth behavior로 부드럽게 진행됩니다.

이 예제는 Bootstrap 5.3의 최신 기능을 사용하며, 모든 주요 브라우저에서 호환됩니다.

<div id="list-example" class="list-group">
  <a class="list-group-item list-group-item-action" href="#list-item-1">Item 1</a>
  <a class="list-group-item list-group-item-action" href="#list-item-2">Item 2</a>
  <a class="list-group-item list-group-item-action" href="#list-item-3">Item 3</a>
  <a class="list-group-item list-group-item-action" href="#list-item-4">Item 4</a>
</div>
<div data-bs-spy="scroll" data-bs-target="#list-example" data-bs-root-margin="0px 0px -40%" style="height: 300px; overflow-y: auto;" tabindex="0">
  <h4 id="list-item-1">Item 1</h4>
  <p>...</p>
  <h4 id="list-item-2">Item 2</h4>
  <p>...</p>
</div>

Simple Anchors

Item 1 Item 2 Item 3 Item 4 Item 5

Item 1

간단한 앵커 링크를 사용한 Scrollspy 예제입니다. 이 방법은 최소한의 마크업으로 Scrollspy를 구현할 수 있습니다.

Bootstrap의 nav나 list-group 컴포넌트를 사용하지 않고도, 일반 앵커 링크만으로 Scrollspy를 구현할 수 있습니다. 커스텀 스타일을 적용하기에 유연한 구조입니다.

d-flex와 flex-column 유틸리티 클래스를 사용하여 세로 레이아웃을 만들고, gap-2로 간격을 조절했습니다.

Item 2

두 번째 항목입니다. 각 링크는 p-1과 rounded 클래스로 기본 스타일이 적용되어 있습니다.

SCSS에서 정의한 커스텀 스타일이 적용되어, 호버 시 배경색이 변하고 활성 상태에서는 primary 색상으로 표시됩니다.

이러한 간단한 구조는 미니멀한 디자인이나 사이드바가 필요 없는 경우에 적합합니다.

커스터마이징이 쉬워서 프로젝트의 디자인 시스템에 맞게 조정할 수 있습니다.

Item 3

세 번째 항목입니다. Scrollspy는 매우 유연한 컴포넌트로, 거의 모든 형태의 네비게이션 구조와 함께 작동합니다.

중요한 것은 data-bs-spy="scroll" 속성이 있는 스크롤 컨테이너와 data-bs-target 속성으로 연결된 네비게이션 요소입니다.

네비게이션의 각 링크는 href 속성으로 콘텐츠의 ID를 참조하면 됩니다. Bootstrap이 나머지를 자동으로 처리합니다.

이 예제는 5개의 항목을 포함하고 있어, 스크롤 동작을 명확하게 확인할 수 있습니다.

Item 4

네 번째 항목입니다. Scrollspy의 data-bs-root-margin 속성을 사용하여 활성화 임계값을 조정할 수 있습니다.

예를 들어 "0px 0px -40%"는 요소가 뷰포트의 하단 40% 지점에 도달했을 때 활성화됨을 의미합니다.

이 설정을 조정하여 사용자 경험을 최적화할 수 있습니다. 일반적으로 -40%에서 -10% 사이의 값이 적절합니다.

tabindex="0" 속성은 키보드로 스크롤 컨테이너에 포커스를 맞출 수 있게 하여 접근성을 향상시킵니다.

Item 5

다섯 번째이자 마지막 항목입니다. 이것으로 Simple Anchors Scrollspy 예제가 완료됩니다.

Scrollspy는 단일 페이지 웹사이트, 제품 랜딩 페이지, 문서 사이트 등에서 널리 사용되는 패턴입니다.

사용자가 현재 어느 섹션을 보고 있는지 시각적 피드백을 제공하여 전체적인 사용자 경험을 개선합니다.

모든 Scrollspy 예제에서 네비게이션 링크를 클릭하면 해당 섹션으로 스크롤되며, 전체 페이지가 아닌 컨테이너 내부만 이동합니다.

Bootstrap 5.3의 Scrollspy는 안정적이고 성능이 우수하며, 다양한 사용 사례를 지원합니다.
